Report Scores
SDK Reporting: If you don't have a backend, you can report scores from the client. This is less secure since a motivated person could submit fake scores with your publishable key. If this is okay with you, make sure you turn ON Client Score Reporting in your dashboard. If you'd prefer to validate scores on your server, turn OFF Client Score Reporting and only submit scores from your server.

Update High Score

Report From SDK

Use high scores when each players aren't directly competing. To report a numeric high score, use the ReportScore(score, optional_leaderboard_id?, player_record_callback?) method. If no leaderboard ID is provided, the first one will be used. Only one leaderboard score can be reported at a time.
galaxyLeaderboard.ReportScore(500.0);

galaxyLeaderboard.ReportScore(24, "leaderboard_002", (newPlayerRecord) => {});
Platform Leaderboards: Reporting scores through the SDK will automatically submit the score to iOS Game Center or Google Play Leaderboard.
Make sure you use the same leaderboard ID on Galaxy and App Store Connect/Google Play Services.

Report Player Scores

Update Skill Rating

Report From SDK

Use Win/Loss when two or more players are playing directly against each other. To report a win or loss, use the UpdateSkill(match_id, [players_ranked]) method. This will allow us to generate an ELO rating for players in your game, which you can use in the future to match similarly skilled players.
Every player in a match should report the outcome using the same match id. We use a consensus based anti-fraud algorithm to prevent cheating.
// Report the outcome of a 3 player game
galaxyLeaderboard.UpdateSkill("match_0001", ["first_place", "second_place", "third_place"]);
Conflicting Data: All players should report outcomes within 10 seconds of each other. If conflicting outcomes are reported from different clients, the entire match will be discarded. If a player reports an outcome more than 10 seconds after the first outcome was received by our servers, it will be ignored.

Report From Server

Send a POST request from your server if you verify outcomes on your backend
